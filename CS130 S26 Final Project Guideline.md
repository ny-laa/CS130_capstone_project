Test Scenarios and Test Cases  
In this section, clearly explain how the proposed application was tested. Describe the individual test scenarios and map them to their corresponding test classes and test methods (e.g., using **JUnit** or another testing framework of your choice). For each test suite, outline the setup and teardown procedures, the specific inputs used in each test case, and the expected outcomes (test oracles). Ensure that every test case description is complete and sufficiently detailed to demonstrate the correctness and robustness of your implementation. Highlight any edge cases, boundary conditions, and error-handling scenarios that were considered.   
The TAs will evaluate the quality of your testing design. Therefore, your documentation should make the structure, intent, and completeness of your tests clear and easy to assess. 

**Include a brief guideline for TAs:**

**1\) To understand how your tests are organized,**   
**2\) And what aspects of the system they validate,**   
**3\) And to install your software and run the tests.** 

You can use GitHub permalinks to the relevant testing infrastructure to facilitate review.

**Describe any software engineering practices your team used during development,** such as version control & pull requests, code reviews, CI/CD pipelines, deployment**.** Extra credit may be given.
